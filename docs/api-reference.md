# HRM API Reference

## Base URL
- `http://<host>/api/v1`

## Local server (port 8001)
Dự án HCM mặc định chạy cổng **8001** (tránh xung đột với dự án khác trên 8000):

```powershell
cd F:\HRM\EHR
php artisan serve --host=127.0.0.1 --port=8001
# hoặc: composer serve
```

- Web app: `http://127.0.0.1:8001/app`
- OpenAPI docs: `http://127.0.0.1:8001/api/v1/docs`

## Authentication
- Current API routes are organized under `/api/v1`
- Authentication uses bearer API tokens.
- Call `POST /api/v1/auth/login` with email/password to receive a token.
- Include `Authorization: Bearer {token}` or `X-API-TOKEN: {token}` on protected requests.
- Seeded roles: `admin`, `hr_manager`, `auditor`.
- Default admin user is seeded with email `admin@example.com` and password `Admin@123`.
- Browse the live OpenAPI docs at `/api/v1/docs` and the raw YAML at `/api/v1/openapi.yaml`.

## Auth Endpoints
- `POST /api/v1/auth/login`
- `GET /api/v1/auth/me`
- `POST /api/v1/auth/logout`
 - `POST /api/v1/auth/rotate` (rotate current user's API token)

Login body:
```json
{
  "email": "admin@example.com",
  "password": "Admin@123"
}
```

Logout requires authentication header.

## Example: use generated admin token

Replace {token} with the token generated by `php artisan user:token admin@example.com`.

curl example:
```bash
curl -H "Authorization: Bearer {token}" \
  -H "Accept: application/json" \
  http://127.0.0.1:8001/api/v1/companies
```

Rotate token (returns new token):
```bash
curl -X POST -H "Authorization: Bearer {token}" \
  -H "Accept: application/json" \
  http://127.0.0.1:8001/api/v1/auth/rotate
```

## Resources

### Companies
- `GET /api/v1/companies`
- `POST /api/v1/companies`
- `GET /api/v1/companies/{company}`
- `PUT /api/v1/companies/{company}`
- `DELETE /api/v1/companies/{company}`

Request body for `POST` / `PUT`:
```json
{
  "name": "HRM Global",
  "code": "COMP-001",
  "tax_code": "HRM2026001",
  "address": "123 Main Street, Hanoi",
  "phone": "+84 24 1234 5678",
  "email": "contact@hrmglobal.local",
  "is_active": true
}
```

Response example (201 Created):
```json
{
  "data": {
    "id": 1,
    "name": "HRM Global",
    "code": "COMP-001",
    "tax_code": "HRM2026001",
    "address": "123 Main Street, Hanoi",
    "phone": "+84 24 1234 5678",
    "email": "contact@hrmglobal.local",
    "is_active": true,
    "created_at": "2026-05-26T12:00:00Z",
    "updated_at": "2026-05-26T12:00:00Z"
  }
}
```

### Audit Logs
- `GET /api/v1/audit-logs`

Audit log entries include actor metadata, entity type, action, and JSON `old_value` / `new_value`.

## Seeders
- Run `php artisan db:seed --class=DatabaseSeeder --force`
- This seeds the default admin account and initial HR sample data.

## Notes
- All request payloads should be sent as JSON.
- Response format uses `{ "data": ... }` for success.
- Validation errors follow Laravel standard JSON error format.
